# RPG Paper Maker — Differences Between Version 2.0 and 3.0

## Overview

Version 3.0 is a major release that completely rewrites the editor from Qt/C++ to a modern React/TypeScript/Electron stack, while also bringing significant new engine features and a new cross-platform web deployment target.

---

## Editor (RPG Paper Maker)

### Complete Rewrite

The editor has been fully rebuilt from scratch in React/TypeScript with Electron. While the workflow is familiar, virtually every screen has been redesigned with improved usability, better performance, and a consistent component library.

### Map Editor

- **Multi-tab map editing** — open and switch between several maps at once.
- **Transformation tools** — move, rotate, and scale 3D objects and map elements directly in the viewport.
  - Rotation buttons to rotate ±90°, with keyboard shortcuts for all transformation modes.
  - Warning indicator when square or pixel mode is active to avoid accidental misalignment.
  - The last placed element is automatically selected after a transformation.
- **Objects list panel** — browse all objects placed on the current map; double-click to open edit mode.

### Mountains

- **Top floor mountains** — you can now designate a mountain as a top-floor element with tileset or autotile.
- **Slopes** — bottom, top, left, and right wall thicknesses are now configured independently (previously a single value applied to all sides).

### 3D Objects

- **GLTF support** — 3D objects can now use GLTF models in addition to OBJ. A dedicated `Shapes/GLTF` folder is created in each project.
- **GLTF move/stop animations** — GLTF objects can be assigned separate animation indices for movement and idle states.

### Data & Systems Windows

- **Save and Close button** — quickly save and dismiss a data or system window with a single click.
- **Cut option** for list items.
- **Paste after** selected item (previously pasted before) and **paste over** option.
- **Edit common reactions inline** — clicking a common reaction dropdown now shows an edit button to jump directly into that reaction's commands.
- **Map names shown** in the "Create Object in Map", "Start Battler", and "Teleport Object" commands.

### Skills

- **"Run on enemy" option** — skills can be configured to have the attacker run toward the target before executing the skill animation. The damage numbers are displayed during the target animation.

### Battle System

- **Battle boss music** — new battle bosse music.

### Commands

- **Disable option** — mark any command as disabled; it will be skipped at runtime without being removed.
- **Script formula textarea** for condition commands.
- **Block hero during reaction** behavior improved.

### Start Position

- **Relocate button** — a dedicated button to reassign the hero's start position on the map without having to place the hero object manually.

### Keyboard

- **AZERTY detection** — the editor auto-detects AZERTY keyboard layouts and adjusts shortcut hints accordingly.

### Previewer

- **Rotation and scale controls** in the 3D previewer for objects.

### Plugins

- Full plugin management UI: install, update, enable/disable, and configure both online (marketplace) and local plugins.

### Deployment

- New **Deploy dialog** with options for web (browser) export and desktop builds.

### Tutorial & Default Project

- A **tutorial project** is included to help new users get started.
- Default project updated with improved tilesets and example content.

### UI Improvements

- Tooltips on dropdowns, lists, and map options show full text when content is truncated (ellipsis).
- Dropdown mousewheel scrolling.
- Draggable dialogs.
- Version update warning bar when the engine is outdated.
- Splitter panels with persistent widths.

## Engine (Game-Scripts)

### GLTF 3D Objects

- 3D objects can now be loaded from GLTF files in addition to OBJ.
- GLTF objects support separate **move** and **stop** animations (by animation index).

### Slopes / Mountains

- Mountains now support **per-direction wall geometry**, enabling slope-like mountain shapes.
- Each side (bottom, top, left, right) can have independent width in squares and pixels.

### Touch / Mobile Input

- The engine now handles **touch events** for mobile browser deployments.

### Battle System

- **Run on enemy** — the attacker physically moves to the target before executing the animation; damage is shown during the target animation phase.
- **Disable commands** — disabled commands are skipped at runtime.
- **Random battle troop selection** — the "start battle with random troop" option now correctly selects a troop from the map's random battles list, weighted by priority.
- Numerous battle fixes: force action, common reaction parameters, damage display, equipment stats, efficiency elements, level up progression.

### DLC Integration

- The engine correctly resolves DLC asset paths at runtime.

### Keyboard

- Key bindings are now stored as **string key names** (e.g. `"ArrowLeft"`, `"Enter"`) instead of Qt integer codes.

### Bug Fixes (notable)

- Fixed reactions in progress being stopped when teleporting to a new map.
- Fixed common reaction parameters propagation.
- Fixed characteristic equip allow/forbid inheritance.
- Fixed division by zero when sprite scale is zero.
- Fixed camera drift on rotation with target offset.
- Fixed floating-point precision issues with variable operations.
- Fixed wall textures and collisions.
- Fixed mountain collision overflows.
- Fixed climb detection.
- Fixed color alpha handling.
- Fixed game-over video not stopping after "continue".
- Fixed save data variables containing stale formula strings.

## Project File Structure Changes (2.0 → 3.0)

The project format changed significantly. The editor's **Project Updater** handles all of these automatically when opening an old project.

| Change                                                                                            | Details                                                   |
| ------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| `Datas/Maps/` → `Maps/`                                                                           | Map folder moved to project root                          |
| `Datas/*.json` → `*.json`                                                                         | All data files moved to project root                      |
| `Datas/treeMap.json` → `treeMaps.json`                                                            | Tree map file renamed                                     |
| `Datas/Scripts/Plugins/` → `Plugins/`                                                             | Plugins folder moved to project root                      |
| `objects.json` merged into `infos.json`                                                           | Per-map object lists consolidated                         |
| `keyBoard.json` → `keyboard.json`                                                                 | Keyboard file renamed, key codes changed to strings       |
| Hero removed from map, `hmp` added to `system.json`                                               | Hero start position now stored in system settings         |
| Default and hero common objects extracted to `do`/`ho` in `commonEvents.json`                     | Cleaner separation of built-in objects                    |
| Map portions: floors/autotiles restructured                                                       | Floors and autotiles unified under a single `lands` array |
| Map portions: sprites kind values decremented by 1                                                | Internal sprite kind enum shifted                         |
| Map portions: walls separated into top-level `walls` array                                        | Walls extracted from sprites                              |
| Map portions: mountains and 3D objects unwrapped                                                  | Removed wrapper object from `moun` and `objs3d`           |
| Troop monsters: `id` → `mid` (monster ID), `id` becomes row index                                 | Cleaner separation of instance ID and monster reference   |
| Statistic progression: duplicate IDs removed                                                      | Classes stat progressions deduplicated                    |
| Formula strings: `Datas.` → `Data.`, `System.` → `Model.`, `Core.Vector2/3` → `THREE.Vector2/3`   | Namespace renames following engine refactor               |
| Title screen commands: unique IDs assigned                                                        | Previously could have duplicate IDs                       |
| Asset placeholders (`id: -1`, `id: 0`): `br: true` added                                          | Marks built-in (BR) assets correctly                      |
| Colors alpha: 0–255 → 0–1                                                                         | Alpha channel normalized                                  |
| Mountain wall: `ws`/`wp` → per-direction (`wsb`, `wst`, `wsl`, `wsr`, `wpb`, `wpt`, `wpl`, `wpr`) | Matches new per-direction mountain feature                |
| `Shapes/GLTF/` folder created                                                                     | Stores GLTF 3D model files                                |
| Special elements: IDs starting at 0 corrected to 1                                                | Autotiles, walls, mountains, objects fixed                |
| Characters picture list: `id: 0` Tileset entry inserted                                           | Ensures correct character picture indexing                |

## Compatibility Notes

### API / Script Formulas

Custom scripts and formulas in your project that reference old namespaces must be updated:

- `Datas.*` → `Data.*`
- `System.*` → `Model.*`
- `Core.Vector2` → `THREE.Vector2`
- `Core.Vector3` → `THREE.Vector3`
- `Core.ReactionInterpreter.currentParameters[N]` → `Core.ReactionInterpreter.currentParameters.get(N)`

The Project Updater applies these renames automatically to all known formula locations. Custom plugin scripts referencing these APIs must be updated manually.

### Saves

Existing save files containing formula strings (e.g. in variables) are migrated by the Project Updater. However, save files from 2.0 cannot be loaded in 3.0 games without running the updater first — they will typically just start a new game gracefully.

### Web Deployment

3.0 introduces first-class web export. Games can be deployed to a browser without additional plugins. Touch input is supported for mobile browsers.
